## 学习重构代码




##### 重构的第一步 确保即将修改的代码拥有一组可靠的测试

>重构前,先检查自己是否具有一套可靠的测试集,这些测试必须有自我校验的能了


一般使用测试工具 jest 

1. 把有关功能的代码抽离成一个单独的函数,按它所干的事情为它命名,把需要引入的变量作为参数传入,把需要返回的值作为返回值返回

2. 修改完后立即执行一次单元测试

3. 测试通过用git提交一次commit

4. 变量名要有意义,不要用a,b,c,d这种

5. 优化掉那些没必要的临时变量

6. 优化函数值给临时变量,把其替换成一个明确的函数

7. 遵循先重构在调优原则
